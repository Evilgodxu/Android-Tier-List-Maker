梯度大师 3.0 后续阶段规划

说明：本规划不含音频录制与动画侧载功能。音频仅保留文件上传绑定方式。

阶段一：时间线构建引擎
目标：根据录制动作、视频配置与音频时长，生成带时间戳的视频时间线。

任务：
1. 解析 actionRecorder.actions，按 tierImages 当前顺序与 actionOrder 生成动作序列。
2. 实现三种编排粒度调度器：
   • 按图片串行：对每张图片完整执行动作顺序后切下一张。
   • 按类型批量：先执行所有图片的第一种动作，再执行第二种，以此类推。
   • 混合模式：根据 VideoGenerationConfig.mixedGranularity 决定每个动作类型是逐图片还是批量。
3. 计算每个动作的开始时间：
   • 图片放置：基础动作耗时（由动画决定，暂定为固定 0.3s）。
   • 图片命名：nameDisplayMode=PER_CHAR 时，耗时 = 名字长度 × nameCharInterval；ONCE 时固定 0.3s。
   • 小图标添加：每个图标耗时 0.3s，同一图片内图标间隔 badgeInterval。
   • 动作类型切换：插入 crossTypePause。
   • 图片切换：图片间隔来源为 AUDIO_DURATION 时，取当前图片 audioUri 时长 + extraAudioOffset；为 FIXED 时取 fixedImageInterval；再叠加 crossImagePause。
4. 输出 Timeline 数据结构（动作时间戳列表、音频段列表、总时长）。

阶段二：视频帧渲染引擎
目标：将时间线逐帧渲染为位图序列，输出 1920×1080。

任务：
1. 创建专用 Composable 帧渲染布局，复用现有 TierList 视觉样式：
   • 标题、作者、层级色块、图片、名称、小图标。
   • 支持与最终保存图片一致的深色/浅色主题。
2. 使用 Compose Recomposer + AndroidComposeView 将 Composable 离屏渲染为 Bitmap：
   • 参考现有 generateTierListBitmap 实现。
   • 按 outputWidth/outputHeight 创建固定尺寸 Canvas。
3. 帧渲染器根据 Timeline 当前时间决定可见元素：
   • 已放置的图片保持显示。
   • 命名动作播放时逐步显示文字（PER_CHAR 模式）。
   • 小图标动作播放时逐个显示。
4. 输出 FPS 默认 30，关键帧按需生成（仅动作变化时生成新帧，静态停留期复用最后一帧）。

阶段三：音频合成引擎
目标：将解说音频、背景音乐、动作音效混音为单条音轨。

任务：
1. 读取每张图片的 audioUri，按 Timeline 中该图片开始放置的时间点对齐播放。
2. 背景音乐（可选）：
   • 从用户选择的 backgroundMusicUri 读取。
   • 循环播放直至视频结束。
   • 音量由 backgroundMusicVolume 控制。
3. 动作音效（可选）：
   • 为图片放置、命名、小图标添加提供简短默认音效（应用内置资源）。
   • audioOverlayMode=REPLACE 时静音动作音效；=KEEP_SFX 时保留；=OVERLAY 时保留并叠加。
   • 音量由 sfxVolume 控制。
4. 使用 MediaCodec + MediaMuxer 或 AudioTrack 进行混音：
   • 推荐先使用 MediaExtractor 解码各音频为 PCM。
   • 按时间轴混合 PCM 样本。
   • 使用 MediaCodec 编码为 AAC，MediaMuxer 封装入 MP4。

阶段四：视频封装与导出
目标：将位图序列与合成音频封装为 MP4（H.264 + AAC）。

任务：
1. 使用 MediaCodec 创建视频编码器（H.264，Surface 输入）：
   • 配置 outputWidth/outputHeight、帧率、码率、关键帧间隔。
   • 使用 MediaMuxer 混合视频轨与音频轨。
2. 将帧渲染 Surface 提供给编码器，按时间线逐帧送入：
   • 每帧附带 presentationTimeUs。
   • 静态停留期重复送入最后一帧以保证时长准确。
3. 音频轨道从阶段三的 AAC 编码器输出获取，同步写入 Muxer。
4. 导出完成后通知系统媒体库或弹出分享/保存选项。

阶段五：预览与导出 UI
目标：在应用内完成低分辨率预览与最终导出。

任务：
1. 在视频生成设置对话框增加「预览视频」与「导出视频」入口。
2. 预览模式：
   • 生成 480p 或更低分辨率的位图序列，在应用内播放器逐帧播放。
   • 同步播放解说音频，检查音画同步。
3. 导出模式：
   • 显示进度对话框（渲染帧数、编码进度、预计剩余时间）。
   • 支持取消操作。
   • 导出成功后提示保存路径或分享。
4. 导出过程在后台协程/Service 中执行，避免 ANR。

阶段六：编排预设管理
目标：将视频生成配置保存为独立预设，与榜单数据解耦。

任务：
1. 新增 VideoPresetManager，管理 JSON 格式的编排预设文件：
   • 存储位置：应用私有目录 /VideoPresets。
   • 包含 actionOrder、granularity、mixedGranularity、所有间隔参数、音频叠加方式、音量、分辨率等。
   • 不包含具体 tierImages 或音频 URI，保持数据无关。
2. 在视频生成设置对话框内增加：
   • 保存当前配置为编排预设。
   • 加载已有编排预设。
   • 重命名、删除、分组、收藏。
3. 导出/导入编排预设（.tdv 或 .json 文件），便于分享。
4. 加载编排预设后，自动按当前 tierImages 顺序与现有音频 URI 生成时间线。

阶段七：性能优化与测试
目标：确保中低端设备也能流畅生成 1080p 视频。

任务：
1. 帧缓存：已渲染帧存入内存或磁盘缓存，静态期复用。
2. 异步流水线：渲染、编码、混音并行执行，使用生产者-消费者队列。
3. 内存优化：避免一次性加载所有帧，按批次处理。
4. 测试场景：
   • 10 张图片、每张 3 个小图标、全部命名、每张 3 秒音频。
   • 验证总时长、音频对齐、动作顺序、文件大小。
5. 错误处理：
   • 音频文件损坏/无法解码时回退到固定间隔。
   • 编码器不支持时降级分辨率或提示用户。

阶段顺序建议：
阶段一 → 阶段二 → 阶段五（预览） → 阶段三 → 阶段四 → 阶段五（导出） → 阶段六 → 阶段七

风险点：
• MediaCodec 在不同厂商设备上的行为差异，需充分测试。
• 大分辨率 + 长视频可能导致 OOM，需尽早实现分帧处理。
• 音频解码与混音精度决定音画同步质量。
